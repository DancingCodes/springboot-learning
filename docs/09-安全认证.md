# Spring Security 安全认证

## 没有安全框架时

每个接口都要手动检查登录状态，几十个方法里到处复制粘贴：

```java
@PostMapping
public String add(@Valid @RequestBody User user, HttpSession session) {
    if (session.getAttribute("user") == null) {
        return "请先登录";
    }
    // 业务逻辑
    return "新增成功";
}
```

## Spring Security 做了什么

在你的 Controller 前面放了一堵墙，所有请求先经过它，放行后才到 Controller：

```
请求 → Security 过滤器链 → 认证 + 授权 → Controller
                        ↓ 失败
                    401 / 403
```

## 认证 vs 授权

| | 认证（Authentication） | 授权（Authorization） |
|---|---|---|
| 问题 | 你是谁？ | 你能干什么？ |
| 失败 | 401 | 403 |
| 例子 | 输入密码登录 | admin 能删用户，user 不能 |

## 三个核心 Bean

配置 Spring Security 就是定义这三个：

### 1. PasswordEncoder — 密码怎么存

```java
@Bean
public PasswordEncoder passwordEncoder() {
    return PasswordEncoderFactories.createDelegatingPasswordEncoder();
}
```

`DelegatingPasswordEncoder` 根据密码前缀选择编码器：`{bcrypt}` 用 BCrypt，`{noop}` 明文。存进数据库的密码形如 `{bcrypt}$2a$10$...`。

### 2. UserDetailsService — 用户从哪来

用户存数据库 `sys_user` 表，不再用内存。启动时 `DataInitializer` 自动插入 admin 用户：

```java
@Service
public class SysUserDetailsService implements UserDetailsService {
    @Override
    public UserDetails loadUserByUsername(String username) {
        SysUser user = sysUserMapper.selectOne(
            new LambdaQueryWrapper<SysUser>().eq(SysUser::getUsername, username));
        if (user == null) throw new UsernameNotFoundException("用户不存在");
        return User.withUsername(user.getUsername())
                .password(user.getPassword())
                .build();
    }
}
```

`SysUser` 是纯实体，不实现 `UserDetails`。返回的是 Spring Security 内置的 `User` 对象（一行 `User.withUsername()` 搞定）。项目只有一个用户，没用到 roles。

### 3. SecurityFilterChain — 拦截规则 + JWT + 401

```java
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/auth/login").permitAll()
            .requestMatchers("/doc.html", "/v3/api-docs/**", "/webjars/**").permitAll()
            .requestMatchers("/file/download").permitAll()  // img 标签不带 token
            .anyRequest().authenticated()
        )
        .sessionManagement(session -> session
            .sessionCreationPolicy(SessionCreationPolicy.STATELESS)  // 无状态，不用 cookie
        )
        .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)
        .csrf(AbstractHttpConfigurer::disable)
        .exceptionHandling(ex -> ex
            .authenticationEntryPoint((request, response, authException) -> {
                response.setStatus(401);
                response.setContentType("application/json");
                objectMapper.writeValue(response.getOutputStream(),
                    Result.error(401, "未登录或token已过期"));
            })
        );
    return http.build();
}
```

**关键决策：**
- 无状态会话（JWT 不靠 cookie），关 CSRF
- JWT Filter 插在 Spring 内置认证前面，只负责"认人"不拦人
- 自定义 401 返回统一 JSON 格式
- `@EnableMethodSecurity` / `@PreAuthorize` 已去掉，项目只有一个用户不需要角色控制

## 认证链路

```
请求 → JwtAuthFilter（识别 token，塞 SecurityContext）
     → authorizeHttpRequests（真正拦人：没认证→401）
     → Controller
```

## 架构简图

```
SecurityConfig
  ├── PasswordEncoder         ← 密码加密（Delegating）
  ├── SysUserDetailsService   ← 用户来源（数据库 sys_user 表）
  └── SecurityFilterChain     ← 拦截规则
        ├── authorizeHttpRequests  → 路径权限（默认全保，例外放行）
        ├── sessionManagement      → 无状态
        ├── addFilterBefore        → JWT Filter
        └── exceptionHandling      → 自定义 401 JSON
```
