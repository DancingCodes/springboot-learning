# Spring Security 安全认证

## 没有安全框架时

每个接口都要手动检查登录状态，几十个方法里到处复制粘贴：

```java
@PostMapping
public String add(@Valid @RequestBody User user, HttpSession session) {
    if (session.getAttribute("user") == null) {
        return "请先登录";
    }
    // 业务逻辑
    return "新增成功";
}
```

## Spring Security 做了什么

在你的 Controller 前面放了一堵墙，所有请求先经过它，放行后才到 Controller：

```
请求 → Security 过滤器链 → 认证 + 授权 → Controller
                        ↓ 失败
                    401 / 403
```

## 认证 vs 授权

| | 认证（Authentication） | 授权（Authorization） |
|---|---|---|
| 问题 | 你是谁？ | 你能干什么？ |
| 失败 | 401 | 403 |
| 例子 | 输入密码登录 | admin 能删用户，user 不能 |

## 三个核心 Bean

配置 Spring Security 就是定义这三个：

### 1. PasswordEncoder — 密码怎么存

```java
import org.springframework.context.annotation.Bean;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.crypto.password.PasswordEncoder;

@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();  // "admin123" → "$2a$10$NqG..." 不可逆
}
```

### 2. UserDetailsService — 用户从哪来

```java
import org.springframework.context.annotation.Bean;
import org.springframework.security.core.userdetails.User;
import org.springframework.security.core.userdetails.UserDetailsService;
import org.springframework.security.provisioning.InMemoryUserDetailsManager;

@Bean
public UserDetailsService userDetailsService() {
    var admin = User.builder()
            .username("admin")
            .password(passwordEncoder().encode("admin123"))
            .roles("ADMIN")
            .build();
    var user = User.builder()
            .username("user")
            .password(passwordEncoder().encode("user123"))
            .roles("USER")
            .build();
    return new InMemoryUserDetailsManager(admin, user);
}
```

学习阶段用内存用户理解机制，生产环境换成数据库只需改这一个 Bean。

### 3. SecurityFilterChain — 哪些路径要保护

```java
import org.springframework.context.annotation.Bean;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.web.SecurityFilterChain;

@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/auth/login").permitAll()  // 登录公开
            .anyRequest().authenticated()                // 其余都要认证
        )
        .httpBasic(withDefaults())
        .csrf(csrf -> csrf.disable());  // REST API 不依赖 cookie，关闭 CSRF
    return http.build();
}
```

安全原则：默认全部受保护，只显式放行少数公开端点。角色控制用 `@PreAuthorize`：

```java
import org.springframework.security.access.prepost.PreAuthorize;

@DeleteMapping("/{id}")
@PreAuthorize("hasRole('ADMIN')")
public Result<Void> delete(@PathVariable Long id) { ... }
```

需要 `@EnableMethodSecurity` 开启。

## HTTP Basic 认证

```java
import static org.springframework.security.config.Customizer.withDefaults;

http.httpBasic(withDefaults());
```

客户端在请求头里带 Base64 编码的用户名密码：

```
Authorization: Basic YWRtaW46YWRtaW4xMjM=   ← admin:admin123 的 Base64
```

## 架构简图

```
SecurityConfig
  ├── PasswordEncoder        ← 密码加密
  ├── UserDetailsService     ← 用户来源
  └── SecurityFilterChain    ← 拦截规则
        ├── authorizeHttpRequests  → 路径权限
        ├── httpBasic              → 认证方式
        └── csrf                   → 关闭跨站防护
```
