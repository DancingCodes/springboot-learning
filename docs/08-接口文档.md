# 接口文档

所有接口统一返回格式：

```json
{
  "code": 200,
  "msg": "成功",
  "data": { ... }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 200 成功，400 参数错误，500 服务器错误 |
| msg | String | 提示信息 |
| data | T | 响应数据，失败时为 null |

---

## 认证 — `/auth`

### 登录

```
POST /auth/login
Content-Type: application/json
```

**请求体**：

```json
{
  "username": "admin",
  "password": "admin123"
}
```

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| username | String | 是 | 不能为空 |
| password | String | 是 | 不能为空 |

**响应**：

```json
{
  "code": 200,
  "msg": "成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9..."
  }
}
```

**失败响应（403）**：用户名或密码错误。

### 后续请求

所有接口（除 `/auth/login`）均需在请求头携带 token：

```
Authorization: Bearer <token>
```

不带或 token 过期返回 403。

## 用户管理 — `/user`

### 1. 分页查询用户

```
GET /user?page=1&size=10
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | int | 否 | 1 | 页码，从 1 开始 |
| size | int | 否 | 10 | 每页条数 |

**响应**：

```json
{
  "code": 200,
  "msg": "成功",
  "data": {
    "records": [
      {
        "id": 1,
        "name": "张三",
        "email": "zhang@example.com",
        "age": 25,
        "createTime": "2026-05-19T10:30:00"
      }
    ],
    "total": 100,
    "size": 10,
    "current": 1,
    "pages": 10
  }
}
```

| data 字段 | 类型 | 说明 |
|-----------|------|------|
| records | Array | 当前页数据 |
| total | long | 总记录数 |
| size | long | 每页条数 |
| current | long | 当前页码 |
| pages | long | 总页数 |

### 2. 查询单个用户

```
GET /user/{id}
```

**响应**：

```json
{
  "code": 200,
  "msg": "成功",
  "data": {
    "id": 1,
    "name": "张三",
    "email": "zhang@example.com",
    "age": 25,
    "createTime": "2026-05-19T10:30:00"
  }
}
```

### 3. 新增用户

```
POST /user
Content-Type: application/json
```

**请求体**：

```json
{
  "name": "张三",
  "email": "zhang@example.com",
  "age": 25
}
```

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| name | String | 是 | 不能为空 |
| email | String | 否 | 合法邮箱格式 |
| age | Integer | 否 | 0 ~ 150 |
| createTime | DateTime | 否 | 不传则数据库自动生成 |

**响应**：

```json
{
  "code": 200,
  "msg": "新增成功",
  "data": {
    "id": 1,
    "name": "张三",
    "email": "zhang@example.com",
    "age": 25,
    "createTime": "2026-05-19T10:30:00"
  }
}
```

**校验失败响应（400）**：

```json
{
  "code": 400,
  "msg": "参数校验失败",
  "data": {
    "name": "姓名不能为空",
    "email": "邮箱格式不正确"
  }
}
```

### 4. 修改用户

```
PUT /user
Content-Type: application/json
```

**请求体**：同新增，需带 `id` 字段。

```json
{
  "id": 1,
  "name": "张三",
  "email": "zhang_new@example.com",
  "age": 26
}
```

**响应**：

```json
{
  "code": 200,
  "msg": "修改成功",
  "data": {
    "id": 1,
    "name": "张三",
    "email": "zhang_new@example.com",
    "age": 26,
    "createTime": "2026-05-19T10:30:00"
  }
}
```

### 5. 删除用户

```
DELETE /user/{id}
```

**响应**：

```json
{
  "code": 200,
  "msg": "删除成功",
  "data": null
}
```

## 账户管理 — `/account`

### 1. 分页查询账户

```
GET /account?page=1&size=10
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | int | 否 | 1 | 页码，从 1 开始 |
| size | int | 否 | 10 | 每页条数 |

**响应**：

```json
{
  "code": 200,
  "msg": "成功",
  "data": {
    "records": [
      {
        "id": 1,
        "name": "张三",
        "balance": 1000.00
      }
    ],
    "total": 50,
    "size": 10,
    "current": 1,
    "pages": 5
  }
}
```

### 2. 转账

```
POST /account/transfer
Content-Type: application/json
```

**请求体**：

```json
{
  "fromId": 1,
  "toId": 2,
  "amount": 100.00
}
```

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| fromId | Long | 是 | 不能为空 |
| toId | Long | 是 | 不能为空 |
| amount | BigDecimal | 是 | 不能为空，必须 ≥ 0.01 |

**响应**：

```json
{
  "code": 200,
  "msg": "转账成功",
  "data": null
}
```

**失败响应（400）**：

```json
{
  "code": 400,
  "msg": "余额不足或账户不存在",
  "data": null
}
```

---

## 通用错误码

| 状态码 | 含义 | 触发场景 |
|--------|------|----------|
| 200 | 成功 | 正常响应 |
| 400 | 请求参数错误 | 校验失败、余额不足、账户不存在 |
| 500 | 服务器内部错误 | 数据库连接失败等未预期异常 |
