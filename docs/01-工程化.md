# 工程化

## Maven vs Gradle

- **Maven**：配置文件 `pom.xml`，老牌主流，教程最多
- **Gradle**：配置文件 `build.gradle`，灵活，Android 和大型项目用

## mvnw / mvnw.cmd — Maven Wrapper

这两个文件让你**不用自己装 Maven** 就能构建项目。

- `mvnw`：Linux / Mac 用
- `mvnw.cmd`：Windows 用

你执行 `mvnw.cmd spring-boot:run` 时，它发现你电脑没装 Maven，就自动下载一个到本地缓存，然后用它来构建项目。别人拿到你的项目也不需要装 Maven，直接跑就行。

## pom.xml — Maven 项目配置文件

pom.xml 是 Maven 的核心，告诉 Maven 三件事：

### 1. 这个项目是谁（基础信息）

```xml
<groupId>com.example</groupId>      <!-- 组织域名倒着写 -->
<artifactId>demo</artifactId>       <!-- 项目名 -->
<version>0.0.1-SNAPSHOT</version>   <!-- 版本号，SNAPSHOT = 开发中 -->
```

### 2. 需要什么依赖（dependencies）

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
</dependencies>
```

Maven 会按 `<groupId>:<artifactId>:<version>` 这个坐标去中央仓库自动下载 jar 包。你不需要手动下载任何东西。

**starter 的含义**：`spring-boot-starter-web` 是一个"启动器"，一个 starter 等于一组相关的 jar 包打包在一起。比如 `starter-web` 里包含了 Spring MVC、内嵌 Tomcat、Jackson（JSON 解析）等等，你不用一个一个去声明。

### 3. 父 POM（继承配置）

```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.5.14</version>
</parent>
```

这行意思是你的项目继承了 Spring Boot 官方的父 POM。父 POM 里已经把常见依赖的版本号定好了，所以你在 `<dependencies>` 里写依赖时不用写 `<version>`，Maven 会自动用父 POM 里定好的版本，避免版本冲突。

### 4. 构建插件（Spring Boot Maven Plugin）

```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>
    </plugins>
</build>
```

作用：把项目打包成一个 **Fat Jar**（也叫 Uber Jar）——一个文件内含了你的代码 + Tomcat + 所有依赖。

- **没有这个插件**：`mvn package` 打出来的 jar 只有你自己的代码，跑不了
- **有这个插件**：`mvn package` 打出来的 jar 包含了所有依赖 + 内嵌 Tomcat，一行命令就能跑：

```bash
java -jar demo-0.0.1-SNAPSHOT.jar
```

## Lombok — 消灭模板代码

### 依赖 + IDEA 插件

```xml
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <optional>true</optional>
</dependency>
```

IDEA: `Settings → Plugins → 搜 Lombok → 安装`

### 常用注解

- `@Data` — = `@Getter` + `@Setter` + `@ToString` + `@EqualsAndHashCode`
- `@AllArgsConstructor` — 生成包含所有字段的构造方法
- `@NoArgsConstructor` — 生成无参构造方法

### 改前 vs 改后

```java
// 改前：29 行，手写所有 getter/setter
@TableName("user")
public class User {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String name;
    private String email;
    private Integer age;
    private LocalDateTime createTime;

    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    // ... 还有 10 行
}

// 改后：14 行，@Data 全搞定
@Data
@TableName("user")
public class User {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String name;
    private String email;
    private Integer age;
    private LocalDateTime createTime;
}
```

### 原理

Lombok 在**编译期**自动生成 getter/setter/toString 等方法到 `.class` 文件里。源码里看不见，但编译后都有，Jackson 和 MyBatis-Plus 照常用。
