## 三层架构

```
Controller → 接收请求，调 Service，返回响应（不含业务逻辑）
    ↓
Service    → 处理业务逻辑，调用 Mapper
    ↓
Mapper     → 只跟数据库交互
```

| | Controller | Service | Mapper |
|---|---|---|---|
| 职责 | 接收请求 | 业务逻辑 | 数据库操作 |
| 注解 | `@RestController` | `@Service` | `@Mapper` |
| 事务 | 不管 | `@Transactional` | 不管 |

## @Transactional — 事务

**抛异常才回滚，不抛就提交。**

```java
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.math.BigDecimal;

@Service
public class AccountService {

    private final AccountMapper accountMapper;

    public AccountService(AccountMapper accountMapper) {
        this.accountMapper = accountMapper;
    }

    @Transactional
    public void transfer(Long fromId, Long toId, BigDecimal amount) {
        int rows = accountMapper.deduct(fromId, amount);
        if (rows == 0) {
            throw new IllegalArgumentException("余额不足或账户不存在");  // 抛异常 → 回滚
        }
        int rows2 = accountMapper.addBalance(toId, amount);
        if (rows2 == 0) {
            throw new IllegalArgumentException("目标账户不存在");       // 抛异常 → 回滚
        }
    }
}
```

流程：

```
1. deduct 扣钱 → SQL 执行成功
2. addBalance 失败 → 返回 0 行
3. throw 异常 → @Transactional 拦截到 → 回滚第 1 步
4. 数据回到调用前状态
```

### 底层原理

MySQL 默认每条 SQL 自动提交。Spring `@Transactional` 帮你关了自动提交，方法正常结束才提交，抛异常就回滚：

```java
import java.sql.Connection;
import javax.sql.DataSource;

// @Transactional 底层相当于：
Connection conn = dataSource.getConnection();
conn.setAutoCommit(false);  // 关闭自动提交
try {
    // 你的业务代码
    conn.commit();           // 成功 → 提交
} catch (Exception e) {
    conn.rollback();         // 异常 → 回滚
}
```

回滚靠的是 MySQL 的 undo log：SQL 确实执行了，数据在内存里也改了，但 undo log 记下了旧值。rollback 时用 undo log 把数据改回去。未提交的修改，其他事务看不到（MVCC）。

### 必须自己写 SQL 的场景

`BaseMapper.updateById` 是全覆盖更新，转账需要在当前值上加减，得自己写：

```java
import org.apache.ibatis.annotations.Mapper;
import org.apache.ibatis.annotations.Param;
import org.apache.ibatis.annotations.Update;
import java.math.BigDecimal;

@Mapper
public interface AccountMapper extends BaseMapper<Account> {

    @Update("UPDATE account SET balance = balance - #{amount} WHERE id = #{id} AND balance >= #{amount}")
    int deduct(@Param("id") Long id, @Param("amount") BigDecimal amount);

    @Update("UPDATE account SET balance = balance + #{amount} WHERE id = #{id}")
    int addBalance(@Param("id") Long id, @Param("amount") BigDecimal amount);

    @Update("UPDATE account SET balance = #{amount} WHERE id = #{id}")
    int setBalance(@Param("id") Long id, @Param("amount") BigDecimal amount);
}
```

| 方法 | SQL | 用途 |
|------|-----|------|
| `deduct` | `balance = balance - #{amount}` | 扣款，`WHERE balance >= #{amount}` 防超扣 |
| `addBalance` | `balance = balance + #{amount}` | 加钱 |
| `setBalance` | `balance = #{amount}` | 直接设置余额 |

### 什么时候不该加 @Transactional

单条 SQL 不需要（MySQL 本身保证原子性），纯读不需要，方法里调了外部服务（短信、API）也不加（会长时间占着数据库连接）。

| 场景 | 加事务？ |
|------|---------|
| 多张表写入，需要同时成功 | ✅ 加 |
| 同一张表多个 UPDATE/DELETE | ✅ 加 |
| 单条 INSERT/UPDATE/DELETE | ❌ 不加 |
| 纯 SELECT | ❌ 不加 |
| 方法里调了外部服务 | ❌ 不加 |

### 常见坑

- **未检查返回值**：`UPDATE` 可能更新 0 行（条件不满足），必须检查并抛异常，否则不会回滚
- **checked exception 不回滚**：默认只回滚 `RuntimeException`，`IOException` 等需要 `@Transactional(rollbackFor = Exception.class)`

```java
import com.baomidou.mybatisplus.extension.plugins.pagination.Page;

public Page<User> page(int pageNum, int pageSize) {
    return userMapper.selectPage(new Page<>(pageNum, pageSize), null);
```

`Page<T>` 包含：

| 字段 | 说明 |
|------|------|
| records | 当前页数据 |

## VO 与 DTO — 实体之外的数据对象

### 为什么需要

Entity 和数据库表一一对应，但真实场景中：

- **出参**往往要组合多张表的数据（比如用户列表同时显示账户余额）
- **入参**只应暴露前端该填的字段（不暴露 id、createTime 等后端控制的字段）

把这些需求都塞进 Entity 会让实体臃肿、职责不清。解决方案是用 VO 和 DTO 各司其职：

```
        入参 DTO               出参 VO
  前端 → UserSaveDTO → Controller → UserVO → 前端
              │                   ↑
              ↓                   │
          User 实体 ──── Service ──┘
              │
              ↓
           Mapper → DB
```

### VO（View Object，视图对象）— 出参用

组合多个表的数据，只包含展示需要的字段：

```java
@Data
public class UserVO {
    private Long id;
    private String name;
    private String email;
    private Integer age;
    private String avatar;
    private LocalDateTime createTime;
    private Long accountId;    // 来自 account 表
    private BigDecimal balance; // 来自 account 表
}
```

Service 中批量查账户后，用 Converter 做转换：

```java
// 注入
private final UserConverter userConverter;

public Page<UserVO> page(int pageNum, int pageSize) {
    Page<User> page = userMapper.selectPage(new Page<>(pageNum, pageSize), null);
    // 批量查关联账户
    List<Long> userIds = page.getRecords().stream().map(User::getId).toList();
    Map<Long, Account> accountMap = accountMapper.selectList(
            new LambdaQueryWrapper<Account>().in(Account::getUserId, userIds))
            .stream()
            .collect(Collectors.toMap(Account::getUserId, a -> a));
    // Entity → VO（MapStruct 自动转换）
    List<UserVO> voList = page.getRecords().stream()
            .map(u -> userConverter.toVO(u, accountMap.get(u.getId())))
            .toList();
    // ...
}
```

### DTO（Data Transfer Object，数据传输对象）— 入参用

只暴露前端该填的字段，加校验注解：

```java
@Data
public class UserSaveDTO {
    private Long id;           // 修改时传，新增时不传

    @NotBlank(message = "姓名不能为空")
    private String name;

    @Email(message = "邮箱格式不正确")
    private String email;

    @Min(0) @Max(150)
    private Integer age;

    private String avatar;
}
```

Controller 中注入 Converter，一行完成 DTO → Entity：

```java
@PostMapping
public Result<UserVO> add(@Valid @RequestBody UserSaveDTO request) {
    User user = userConverter.toEntity(request);  // 一行替代 5 个 setter
    userService.add(user);
    return Result.success("新增成功", userService.getById(user.getId()));
}
```

### MapStruct — 自动生成转换代码

手写 `toVO()` 和 `toEntity()` 虽然直观，但字段多了以后全是重复的 setter。MapStruct 在**编译期**自动生成这些代码，效果和手写一样（零反射，高性能）。

**Converter 接口**（`converter/UserConverter.java`）：

```java
@Mapper(componentModel = "spring")  // 生成 Spring Bean，可注入
public interface UserConverter {

    @Mapping(source = "user.id", target = "id")
    @Mapping(source = "user.name", target = "name")
    @Mapping(source = "account.id", target = "accountId")
    @Mapping(source = "account.balance", target = "balance")
    UserVO toVO(User user, Account account);

    @Mapping(target = "createTime", ignore = true)
    User toEntity(UserSaveDTO dto);
}
```

- `@Mapping` — 标注字段名不一致或重名的映射。**多源（两个及以上参数）时，同名字段必须显式指定来源**，否则编译报 `Several possible source properties`
- `ignore = true` — 跳过该字段（`createTime` 由 Service 的 `add()` 设置，不从前端传入）
- `componentModel = "spring"` — 生成的实现类带 `@Component`，可以用 `@RequiredArgsConstructor` 注入

**常见坑：多源歧义** — User 和 Account 都有 `id` 和 `name`，MapStruct 不知道 VO 该取哪个，必须写 `@Mapping(source = "user.id", target = "id")` 指定。单源时（如 `toEntity(UserSaveDTO dto)`）同名字段自动映射，不需要显式声明。

**编译时自动生成的实现类**（在 `target/generated-sources/annotations/` 下），等价于：

```java
@Component
public class UserConverterImpl implements UserConverter {
    public UserVO toVO(User user, Account account) {
        UserVO vo = new UserVO();
        vo.setId(user.getId());              // ← 显式指定 user.id
        vo.setName(user.getName());          // ← 显式指定 user.name
        vo.setEmail(user.getEmail());        // ← 只在 User 中有，自动映射
        vo.setAge(user.getAge());            // ← 同名字段自动拷贝
        vo.setAvatar(user.getAvatar());
        vo.setCreateTime(user.getCreateTime());
        vo.setAccountId(account == null ? null : account.getId());
        vo.setBalance(account == null ? null : account.getBalance());
        return vo;
    }
    // toEntity 同理
}
```

**依赖配置**（pom.xml）：

```xml
<dependency>
    <groupId>org.mapstruct</groupId>
    <artifactId>mapstruct</artifactId>
    <version>1.6.3</version>
</dependency>

<!-- maven-compiler-plugin 中注册注解处理器 -->
<annotationProcessorPaths>
    <path>org.mapstruct:mapstruct-processor:1.6.3</path>
    <path>org.projectlombok:lombok</path>
    <path>org.projectlombok:lombok-mapstruct-binding:0.2.0</path>
</annotationProcessorPaths>
```

`lombok-mapstruct-binding` 确保 Lombok 先运行（生成 getter/setter），MapStruct 后运行（调用 getter/setter），两者不冲突。

### 总结

| 对象 | 全称 | 方向 | 职责 |
|------|------|------|------|
| Entity | — | ↔ 持久层 | 和数据库表一一映射，字段 = 列 |
| VO | View Object | ← 出参 | 组合多个表的数据，给前端展示 |
| DTO | Data Transfer Object | → 入参 | 限定前端能传的字段，加校验 |

**核心原则：Entity 只映射自己的表，VO 管展示，DTO 管输入，各司其职。**
