# 拦截器

## Filter vs Interceptor

| | Filter | Interceptor |
|------|--------|-------------|
| 层级 | Servlet 容器 | Spring 容器 |
| 能注入 Bean | 不能直接用 | 可以 |
| 知道目标方法 | 不知道 | `handler` 就是 |
| 时机 | 请求进入 Servlet 前 | Controller 前 → 后 → 视图渲染后 |

简单记：Filter 管"能不能进"（认证），Interceptor 管"进去之后干什么"（日志、审计）。

## 代码

```java
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import lombok.extern.slf4j.Slf4j;
import org.springframework.web.servlet.HandlerInterceptor;

@Slf4j
public class LogInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                             Object handler) {
        request.setAttribute("startTime", System.currentTimeMillis());
        log.info("{} {} → {}", request.getMethod(), request.getRequestURI(), handler);
        return true;  // true = 放行，false = 拦截
    }

    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response,
                           Object handler, ModelAndView modelAndView) {
        log.info("状态码: {}", response.getStatus());
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                Object handler, Exception ex) {
        long startTime = (long) request.getAttribute("startTime");
        log.info("耗时: {}ms", System.currentTimeMillis() - startTime);
        if (ex != null) {
            log.error("异常: {}", ex.getMessage());
        }
    }
}
```

三个时机：
- `preHandle` — Controller 执行前，返回 `false` 则终止请求
- `postHandle` — Controller 执行后，能拿到状态码
- `afterCompletion` — 视图渲染后，**无论如何都会执行**（即使抛异常）

## 注册

```java
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.InterceptorRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new LogInterceptor())
                .addPathPatterns("/**")
                .excludePathPatterns("/doc.html", "/v3/api-docs/**", "/webjars/**");
    }
}
```

## 和 @Aspect 对比

| | AOP | Interceptor |
|------|-----|-------------|
| 粒度 | 方法级（切任意 Service） | Controller 级 |
| 能拿到的 | 参数值、返回值、异常 | request/response、状态码 |
| 优势 | 更细粒度 | `afterCompletion` 保证执行 |

实际项目中通常：Filter（认证）→ Interceptor（日志/审计）→ AOP（业务日志/事务）。
