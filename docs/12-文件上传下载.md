# 对象存储（COS + CDN）

文件存储从本地磁盘迁移到腾讯云 COS，CDN 加速访问。

## 架构

```
用户 ─上传─→ 后端 ─SDK─→ COS (私有读写)
用户 ─访问─→ CDN ─回源鉴权─→ COS
用户 ─下载─→ 后端 ─SDK─→ COS → 返回文件流
```

- **COS Bucket**：私有读写，外部无法直接访问
- **CDN**：HTTPS 证书自动续期，回源鉴权读取 COS
- **下载**：走后端，COS SDK 取流，`Content-Disposition: attachment` 触发浏览器下载

## 配置

```yaml
# application.yml（默认值）
cos:
  secret-id: ${COS_SECRET_ID:}
  secret-key: ${COS_SECRET_KEY:}
  region: ap-beijing
  bucket: springboot-learning-1303900201
  cdn-domain: https://cdn.moonc.love
```

密钥放在 `application-local.yml`，不提交 git。

## CosService

封装 COS SDK 操作：

| 方法 | 作用 |
|------|------|
| `upload(MultipartFile)` | 上传文件，返回 COS key（UUID_原名） |
| `delete(String key)` | 从 COS 删除 |
| `getObject(String key)` | 获取文件输入流（用于下载） |
| `getCdnUrl(String key)` | 拼接 CDN 访问地址 |

```java
@Service
public class CosService {
    // @PostConstruct 初始化 COSClient
    // @PreDestroy 关闭 COSClient
    // 上传时通过 ObjectMetadata 设置 Content-Type
}
```

## 上传

```java
@PostMapping("/upload")
public Result<FileVO> upload(@RequestParam("file") MultipartFile file) {
    String key = cosService.upload(file);
    FileVO vo = new FileVO();
    vo.setOriginalName(file.getOriginalFilename());
    vo.setStoredName(key);       // COS key，存数据库
    vo.setUrl(cosService.getCdnUrl(key)); // CDN 完整地址
    vo.setSize(file.getSize());
    return Result.success(vo);
}
```

返回 `FileVO`（originalName、storedName、url、size）。`storedName` 存数据库，`url` 直接用于前端 `<img>` 展示。

## 下载

```java
@GetMapping("/download")
public ResponseEntity<Resource> download(@RequestParam String storedName) {
    String originalName = storedName.substring(storedName.indexOf("_") + 1);
    Resource resource = new InputStreamResource(cosService.getObject(storedName));
    return ResponseEntity.ok()
            .contentType(MediaType.APPLICATION_OCTET_STREAM)
            .header("Content-Disposition",
                    "attachment; filename*=UTF-8''" + URLEncoder.encode(originalName, UTF_8))
            .body(resource);
}
```

只做下载，不搞 inline 展示。展示统一走 CDN。

## 前端使用

后端返回 `UserVO.avatarUrl`（CDN 完整地址），前端直接用：

```html
<img :src="record.avatarUrl" />
```

下载按钮用 `<a>` 标签原生行为，`getDownloadUrl()` 统一生成 URL：

```html
<a-button :href="getDownloadUrl(record.avatar)" :disabled="!record.avatar">下载头像</a-button>
```

## 安全放行

下载接口不需要登录（`<a>` 标签请求不带 JWT）：

```java
.requestMatchers("/file/download").permitAll()
```

## 与本地存储的对比

| 维度 | 之前（本地磁盘） | 现在（COS + CDN） |
|------|-----------------|-------------------|
| 存储位置 | `./uploads/` | 腾讯云 COS |
| 访问方式 | 后端读文件返回 | CDN 直接访问 |
| 下载 | 后端读文件返回 | 后端从 COS 取流返回 |
| 跨环境 | 部署丢文件 | 云存储持久 |
| 性能 | 单机带宽 | CDN 加速 |
