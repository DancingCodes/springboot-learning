# 文件上传下载

## 为什么

Web 应用常见的文件操作：头像上传、附件下载。Spring 提供了 `MultipartFile`（上传）和 `Resource`（下载）两个核心抽象。

## 上传

```java
@PostMapping("/upload")
public Result<Map<String, String>> upload(@RequestParam("file") MultipartFile file) throws IOException {
    String storedName = UUID.randomUUID().toString() + "_" + file.getOriginalFilename();
    Path target = uploadDir.resolve(storedName);
    file.transferTo(target.toFile());
    return Result.success("上传成功", Map.of("storedName", storedName));
}
```

**流程**：`MultipartFile` → `UUID_原始名` 防重名 → `transferTo()` 写磁盘。

## 下载

```java
@GetMapping("/download/{storedName}")
public ResponseEntity<Resource> download(@PathVariable String storedName) {
    Resource resource = new FileSystemResource(uploadDir.resolve(storedName));
    if (!resource.exists()) return ResponseEntity.notFound().build();

    return ResponseEntity.ok()
            .contentType(MediaType.APPLICATION_OCTET_STREAM)
            .header(HttpHeaders.CONTENT_DISPOSITION,
                    "attachment; filename*=UTF-8''" + URLEncoder.encode(originalName, StandardCharsets.UTF_8))
            .body(resource);
}
```

**关键点**：
- `ResponseEntity<Resource>` — 文件下载的标准返回类型
- `Content-Disposition: attachment` — 告诉浏览器"下载"而不是"预览"
- `filename*=UTF-8''` + `URLEncoder.encode` — 中文文件名不乱码

## 配置

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB      # 单个文件上限
      max-request-size: 10MB   # 单次请求总上限

file:
  upload-dir: ./uploads        # 上传目录
```

## Knife4j 测试文件上传

Knife4j 支持 `@RequestParam MultipartFile`，页面会出现文件选择按钮，直接点选文件上传。

## 核心概念

### MultipartFile
Spring 对上传文件的封装，常用方法：
- `getOriginalFilename()` — 原始文件名
- `getSize()` — 文件大小（字节）
- `transferTo(File)` — 直接写入磁盘
- `getInputStream()` — 读文件流（大数据用流式处理）

### Resource
Spring 对资源的抽象，不论文件来自磁盘、classpath、还是网络，统一用 `Resource` 读。这里用 `FileSystemResource` 因为文件在磁盘上。

### URLEncoder
浏览器解析 `Content-Disposition` 头中的文件名默认用 ISO-8859-1，中文会乱码。`filename*=UTF-8''` 是 RFC 5987 标准，让浏览器按 UTF-8 解码。
