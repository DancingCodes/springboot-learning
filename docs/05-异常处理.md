# 统一异常处理

## 没有异常处理时

Service 里抛了异常，浏览器看到的是 Spring 默认错误页，500 状态码 + 堆栈信息，既丑又**不安全**（内部类名、方法名全泄露）。

## 有了统一异常处理后

返回结构清晰、安全的 JSON，由前端拦截器统一弹窗提示。

## 代码结构

```
exception/
└── GlobalExceptionHandler.java （全局异常拦截器）
dto/
└── Result.java                 （统一响应格式：code + msg + data）
```

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    // 校验异常：@Valid 校验失败时触发，把所有字段错误拼成一句话
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<Void> handleValidation(MethodArgumentNotValidException ex) {
        String msg = ex.getBindingResult().getFieldErrors().stream()
                .map(e -> e.getField() + "：" + e.getDefaultMessage())
                .collect(Collectors.joining("；"));
        return Result.error(500, msg);
    }

    // 兜底：所有其他未处理的异常，直接把异常消息返回
    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<Void> handleOtherException(Exception e) {
        return Result.error(500, e.getMessage());
    }
}
```

## 设计思路：为什么统一返回 500

前端用 axios 拦截器统一处理非 200 响应，`message.error(msg)` 弹窗提示。后端不需要区分 400/500，把错误信息清晰放进 `msg` 即可，前端负责展示。

业务代码抛异常时，把原因写进 message：

```java
throw new IllegalArgumentException("余额不足");
```

前端用户就会看到弹窗："余额不足"。

## MethodArgumentNotValidException

当 `@Valid` 校验失败时，Spring 不会进入 Controller 方法，而是直接抛出 `MethodArgumentNotValidException`。通过 `getBindingResult().getFieldErrors()` 拿到每个字段的错误信息，拼成 `"name：姓名不能为空；email：邮箱格式不正确"` 这种一目了然的中文提示。

## 三个核心注解

- `@RestControllerAdvice` — 全局异常拦截器，对**所有** Controller 生效
- `@ExceptionHandler(Xxx.class)` — 只拦截指定的异常类型
- `@ResponseStatus(HttpStatus.XXX)` — 控制 HTTP 响应的状态码

## 异常匹配顺序

Spring 按**从具体到模糊**匹配：

1. 先尝试 `MethodArgumentNotValidException.class`（更具体）
2. 都没命中则由 `Exception.class` 兜底（所有异常的父类）

## Java 异常体系（简化）

```
Throwable
  ├── Error（JVM 级灾难，不用管）
  └── Exception
        ├── RuntimeException
        │     ├── IllegalArgumentException   ← 参数不合法
        │     ├── NullPointerException       ← 空指针
        │     └── MethodArgumentNotValidException ← @Valid 校验失败
        └── IOException ← 编译时异常，必须 try-catch
```

## HTTP 状态码

| 状态码 | 含义 | 例子 |
|--------|------|------|
| 200 | 成功 | OK |
| 500 | 服务器出错 | 参数校验失败、数据库挂了 |
