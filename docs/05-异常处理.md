# 统一异常处理

## 没有异常处理时

Service 里抛了异常，浏览器看到的是 Spring 默认错误页，500 状态码 + 堆栈信息，既丑又**不安全**（内部类名、方法名全泄露）。

## 有了统一异常处理后

返回结构清晰、安全的 JSON，状态码也正确（参数错 400，服务器错 500）。

## 代码结构

```
exception/
├── ErrorResponse.java          （统一错误格式：code + message）
└── GlobalExceptionHandler.java （全局异常拦截器）
```

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    // 业务异常：参数不合法（如余额不足）
    @ExceptionHandler(IllegalArgumentException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ErrorResponse handleIllegalArgument(IllegalArgumentException e) {
        return new ErrorResponse(400, e.getMessage());
    }

    // 校验异常：@Valid 校验失败时自动触发
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public Map<String, Object> handleValidation(MethodArgumentNotValidException ex) {
        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getFieldErrors().forEach(e ->
            errors.put(e.getField(), e.getDefaultMessage())
        );
        // 返回: {"code":400, "msg":"参数校验失败", "errors":{"name":"姓名不能为空"}}
    }

    // 兜底：所有其他未处理的异常
    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public ErrorResponse handleOtherException(Exception e) {
        return new ErrorResponse(500, "服务器内部错误");
    }
}
```

## MethodArgumentNotValidException

当 `@Valid` 校验失败时，Spring 不会进入 Controller 方法，而是直接抛出 `MethodArgumentNotValidException`。通过 `ex.getBindingResult().getFieldErrors()` 可以拿到每个字段的具体错误信息。

## 三个核心注解

- `@RestControllerAdvice` — 全局异常拦截器，对**所有** Controller 生效
- `@ExceptionHandler(Xxx.class)` — 只拦截指定的异常类型
- `@ResponseStatus(HttpStatus.XXX)` — 控制 HTTP 响应的状态码

## 异常匹配顺序

Spring 按**从具体到模糊**匹配：

1. 先尝试精确匹配（如 `IllegalArgumentException.class`）
2. 然后尝试 `MethodArgumentNotValidException.class`
3. 都没命中则由 `Exception.class` 兜底（所有异常的父类）

## Java 异常体系（简化）

```
Throwable
  ├── Error（JVM 级灾难，不用管）
  └── Exception
        ├── RuntimeException
        │     ├── IllegalArgumentException   ← 参数不合法
        │     ├── NullPointerException       ← 空指针
        │     └── MethodArgumentNotValidException ← @Valid 校验失败
        └── IOException ← 编译时异常，必须 try-catch
```

## HTTP 状态码

| 状态码 | 含义 | 例子 |
|--------|------|------|
| 200 | 成功 | OK |
| 400 | 客户端请求有问题 | 参数为空、校验失败 |
| 500 | 服务器内部出错 | 数据库挂了 |
