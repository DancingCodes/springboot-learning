# 统一异常处理

## 没有异常处理时

Service 抛了异常，浏览器看到的是 Spring 默认错误页：500 + 堆栈信息。既丑又不安全（内部类名、方法名全暴露）。

## 统一返回格式 — Result

```java
public class Result<T> {
    private int code;
    private String msg;
    private T data;

    public static <T> Result<T> success(T data) {
        Result<T> r = new Result<>();
        r.code = 200;
        r.msg = "success";
        r.data = data;
        return r;
    }

    public static <T> Result<T> error(int code, String msg) {
        Result<T> r = new Result<>();
        r.code = code;
        r.msg = msg;
        return r;
    }

    // getter / setter 省略
}
```

## 全局异常拦截器

```java
import org.springframework.http.HttpStatus;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.ResponseStatus;
import org.springframework.web.bind.annotation.RestControllerAdvice;
import java.util.stream.Collectors;

@RestControllerAdvice  // 对所有 Controller 生效
public class GlobalExceptionHandler {

    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<Void> handleValidation(MethodArgumentNotValidException ex) {
        String msg = ex.getBindingResult().getFieldErrors().stream()
                .map(e -> e.getField() + "：" + e.getDefaultMessage())
                .collect(Collectors.joining("；"));
        return Result.error(500, msg);
    }

    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<Void> handleOtherException(Exception e) {
        return Result.error(500, e.getMessage());
    }
}
```

## 工作流程

```
Controller 方法执行
    ↓ 抛异常
@RestControllerAdvice 拦截
    ↓ 匹配 @ExceptionHandler
返回 Result JSON（code=500, msg=具体错误信息）
    ↓
前端 axios 拦截器拿到非 200 → message.error(msg) 弹窗提示
```

业务代码只需把原因写进异常消息：

```java
throw new IllegalArgumentException("余额不足");
```

前端用户就会看到弹窗："余额不足"。

## Spring 异常匹配顺序

从具体到模糊：

1. 先匹配 `MethodArgumentNotValidException`（具体）
2. 没命中则由 `Exception` 兜底（所有异常的父类）

## Java 异常体系（简化）

```
Throwable
  ├── Error（JVM 级，不用管）
  └── Exception
        ├── RuntimeException
        │     ├── IllegalArgumentException
        │     ├── NullPointerException
        │     └── MethodArgumentNotValidException  ← @Valid 校验失败
        └── IOException  ← 编译时异常，必须 try-catch
```
