# 异步任务

## 解决什么问题

转账操作包含三件事：扣钱、加钱、写日志。写日志不是转账的核心流程——用户不等日志写完才看到"转账成功"，但写日志需要一次数据库 IO。

同步写法：扣钱→加钱→写日志→返回。写日志的 IO 直接延长了转账响应时间。

异步写法：扣钱→加钱→触发异步写日志→立刻返回。日志在另一个线程慢慢写，转账请求不等它。

## 核心两个注解

| 注解 | 位置 | 作用 |
|------|------|------|
| `@EnableAsync` | 启动类 | 开启异步支持，背后是线程池 |
| `@Async` | 方法上 | 方法在独立的线程里执行，调用方不等待 |

## 实战：转账日志异步写入

### 建表

```sql
CREATE TABLE IF NOT EXISTS transfer_log (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    from_account_id BIGINT,
    to_account_id BIGINT,
    amount DECIMAL(10, 2),
    create_time DATETIME
);
```

### 启动类加 @EnableAsync

```java
@SpringBootApplication
@EnableScheduling
@EnableAsync
public class SpringbootLearningApplication { ... }
```

### TransferLogService — @Async 异步写

```java
@Slf4j
@Service
@RequiredArgsConstructor
public class TransferLogService {

    private final TransferLogMapper transferLogMapper;

    @Async
    public void logAsync(Long fromAccountId, Long toAccountId, BigDecimal amount) {
        TransferLog log = new TransferLog();
        log.setFromAccountId(fromAccountId);
        log.setToAccountId(toAccountId);
        log.setAmount(amount);
        log.setCreateTime(LocalDateTime.now());
        transferLogMapper.insert(log);
    }
}
```

### AccountService — 转账后触发

```java
@Transactional
public void transfer(Long fromId, Long toId, BigDecimal amount) {
    accountMapper.deduct(fromId, amount);
    accountMapper.addBalance(toId, amount);
    transferLogService.logAsync(fromId, toId, amount);  // 异步，不等它写完
}
```

## 同步 vs 异步

```
同步调用：
  transfer() ──→ logService.log() ──→ 写数据库
       ↑                                  │
       └──────── 等它写完 ←───────────────┘

异步调用：
  transfer() ──→ logService.logAsync() ──→ 立即返回
       │                  │
       └── 继续执行       └── 另一个线程慢慢写数据库（不阻塞主线程）
```

用 `@Async` 后，Spring 返回一个代理对象。调用 `logAsync()` 时，代理把任务丢给线程池，方法立即返回。主线程继续（事务提交、返回响应），线程池里的某个线程执行写日志。

## 为什么日志不放在事务里

日志是审计记录，不是核心业务。日志写失败不应该导致转账回滚——转账成功是最重要的，日志丢了可以补。`@Async` 让日志跑在事务外、跑在另一个线程，隔离了两者的成败。

但这也意味着：转账事务提交了，异步线程可能读不到最新数据（如果读的话）。此处只写不读，不受影响。

## 验证异步

启动后执行一次转账，观察日志时间戳。

如果 AOP 日志里 `→` 和 `←` 之间耗时极短（比如 2ms），说明转账本体没有等日志写完就返回了。

也可以让 `logAsync` 里 `sleep(10000)` 故意慢 10 秒，检查转账接口是否被拖慢——用 `@Async` 后接口应该秒回，10 秒后日志才出现在数据库。
