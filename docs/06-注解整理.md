# 项目注解整理

## Spring Boot / Spring MVC

### `@SpringBootApplication`

启动类标记，= `@Configuration` + `@EnableAutoConfiguration` + `@ComponentScan`。

```java
@SpringBootApplication
public class SpringbootLearningApplication {
    public static void main(String[] args) {
        SpringApplication.run(SpringbootLearningApplication.class, args);
    }
}
```

### `@RestController`

相当于 `@Controller` + `@ResponseBody`，方法返回值直接写入响应体（JSON/字符串），不用每个方法都写 `@ResponseBody`。

```java
@RestController
public class UserController {
    // 所有方法返回的字符串/对象都会直接写入 Response Body
}
```

### `@RequestMapping`

类级别 URL 前缀，该类所有接口都以该路径开头。

```java
@RestController
@RequestMapping("/user")   // 下面所有接口都以 /user 开头
public class UserController { }
```

### `@GetMapping`

映射 HTTP GET 请求。可带路径或空（匹配类前缀）。

```java
@GetMapping                // GET /user?page=1&size=10 → 分页查询
public Result<Page<User>> list(@RequestParam(defaultValue = "1") int page,
                               @RequestParam(defaultValue = "10") int size) { ... }

@GetMapping("/{id}")       // GET /user/1 → 查单个
public Result<User> getById(@PathVariable Long id) { ... }
```

### `@RequestParam`

从 URL 查询参数（`?key=value`）中取值，注入方法参数。

```java
// GET /user?page=2&size=20  →  page=2, size=20
public Result<Page<User>> list(@RequestParam(defaultValue = "1") int page,
                               @RequestParam(defaultValue = "10") int size) { ... }
```

`defaultValue` 指定参数缺省时的默认值。

### `@PostMapping`

映射 HTTP POST 请求，通常用于新增或提交操作。

```java
@PostMapping               // POST /user → 新增用户
public String add(@Valid @RequestBody User user) {
    userService.add(user);
    return "新增成功";
}

@PostMapping("/transfer")  // POST /account/transfer → 转账
public String transfer(@Valid @RequestBody TransferRequest req) { ... }
```

### `@Valid`

触发 Jakarta Validation 校验，加在 Controller 方法的 `@RequestBody` 参数前。校验失败时抛出 `MethodArgumentNotValidException`，方法体不会执行。

```java
@PostMapping
public String add(@Valid @RequestBody User user) { ... }
```

### `@PutMapping`

映射 HTTP PUT 请求，通常用于全量更新。

```java
@PutMapping                // PUT /user → 修改用户
public String update(@Valid @RequestBody User user) {
    userService.update(user);
    return "修改成功";
}
```

### `@DeleteMapping`

映射 HTTP DELETE 请求，通常用于删除。

```java
@DeleteMapping("/{id}")    // DELETE /user/1 → 删除用户
public String delete(@PathVariable Long id) {
    userService.delete(id);
    return "删除成功";
}
```

### `@PathVariable`

从 URL 路径中提取变量值注入方法参数。

```java
@GetMapping("/{id}")
public User getById(@PathVariable Long id) {
    // URL: /user/5  →  id = 5
    return userService.getById(id);
}
```

### `@RequestBody`

将请求体中的 JSON 反序列化为 Java 对象。

```java
@PostMapping
public String add(@RequestBody User user) {
    // 请求体: {"name":"张三","email":"zhang@example.com"}
    // → Jackson 自动创建 User 对象并填充字段
    userService.add(user);
    return "新增成功";
}
```

### `@Service`

声明 Service 层 Bean，交给 Spring 容器管理。

```java
@Service
public class UserService {
    // Spring 扫描到后自动 new 并放入容器
}
```

### `@RestControllerAdvice`

全局异常拦截器，对所有 `@RestController` 生效。一个类搞定整个项目的异常处理。

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    // 这里集中处理所有异常
}
```

### `@ExceptionHandler`

声明拦截哪种异常类型。Spring 按从具体到模糊的顺序匹配。

```java
@ExceptionHandler(IllegalArgumentException.class)  // 精确匹配：业务参数异常
@ResponseStatus(HttpStatus.BAD_REQUEST)            // 返回 400
public Result<Void> handleIllegalArgument(IllegalArgumentException e) {
    return Result.error(400, e.getMessage());
}

@ExceptionHandler(MethodArgumentNotValidException.class) // @Valid 校验失败
@ResponseStatus(HttpStatus.BAD_REQUEST)
public Result<Map<String, String>> handleValidation(MethodArgumentNotValidException ex) {
    // ex.getBindingResult().getFieldErrors() 可拿到每个字段的具体错误
    return Result.error(400, "参数校验失败", errors);
}

@ExceptionHandler(Exception.class)                 // 兜底匹配：所有其他异常
@ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)  // 返回 500
public Result<Void> handleOtherException(Exception e) {
    return Result.error(500, "服务器内部错误");
}
```

### `@ResponseStatus`

控制 HTTP 响应的状态码。

```java
@ResponseStatus(HttpStatus.BAD_REQUEST)           // → 400
@ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)  // → 500
```

### `@Transactional`

开启数据库事务。方法内任意位置抛 `RuntimeException` 时，所有已执行的 SQL 自动回滚。

```java
@Transactional
public void transfer(Long fromId, Long toId, BigDecimal amount) {
    // 第 1 步：扣钱
    int rows = accountMapper.deduct(fromId, amount);
    if (rows == 0) {
        throw new IllegalArgumentException("余额不足");  // 抛异常 → 回滚
    }
    // 第 2 步：加钱
    int rows2 = accountMapper.addBalance(toId, amount);
    if (rows2 == 0) {
        throw new IllegalArgumentException("目标账户不存在"); // 抛异常 → 第 1 步也撤销
    }
}
```

## MyBatis / MyBatis-Plus

### `@Mapper`

声明 MyBatis Mapper 接口，交给 Spring 管理。继承 `BaseMapper<T>` 后自带 CRUD 方法，无需写 SQL。

```java
@Mapper
public interface UserMapper extends BaseMapper<User> {
    // 零 SQL，BaseMapper 自带 selectList / selectById / insert / updateById / deleteById
}
```

### `@TableName`

实体类到数据库表的映射。

```java
@TableName("user")   // 映射到 user 表
public class User { ... }

@TableName("account")  // 映射到 account 表
public class Account { ... }
```

### `@TableId`

标记主键字段，`IdType.AUTO` = 数据库自增。

```java
@TableName("user")
public class User {
    @TableId(type = IdType.AUTO)  // id 由数据库自动生成
    private Long id;
}
```

### `@Update`

声明自定义 UPDATE SQL。当 `BaseMapper.updateById` 不够用时（比如需要在当前值基础上加减），自己写 SQL。

```java
@Mapper
public interface AccountMapper extends BaseMapper<Account> {

    @Update("UPDATE account SET balance = balance - #{amount} WHERE id = #{id} AND balance >= #{amount}")
    int deduct(@Param("id") Long id, @Param("amount") BigDecimal amount);

    @Update("UPDATE account SET balance = balance + #{amount} WHERE id = #{id}")
    int addBalance(@Param("id") Long id, @Param("amount") BigDecimal amount);
}
```

### `@Param`

将方法参数绑定到 SQL 中的 `#{}` 占位符。

```java
@Update("UPDATE account SET balance = balance - #{amount} WHERE id = #{id}")
int deduct(@Param("id") Long id, @Param("amount") BigDecimal amount);
//                ↑                    ↑
//         绑定到 #{id}          绑定到 #{amount}
```

## Lombok

Lombok 在**编译期**生成代码到 `.class` 文件中，源码里看不见，但运行时存在。

### `@Data`

= `@Getter` + `@Setter` + `@ToString` + `@EqualsAndHashCode` + `@RequiredArgsConstructor`。实体类必备。

```java
@Data
@TableName("user")
public class User {
    @TableId(type = IdType.AUTO)
    private Long id;

    @NotBlank(message = "姓名不能为空")
    private String name;

    @Email(message = "邮箱格式不正确")
    private String email;

    @Min(value = 0, message = "年龄不能为负数")
    @Max(value = 150, message = "年龄超出合理范围")
    private Integer age;

    private LocalDateTime createTime;
    // 不用手写任何 getter/setter
}
```

### `@RequiredArgsConstructor`

为 `final` 字段生成构造方法，替代 `@Autowired` 字段注入。

```java
@RestController
@RequestMapping("/user")
@RequiredArgsConstructor
public class UserController {
    private final UserService userService;
    // Lombok 生成: public UserController(UserService userService) { ... }
    // Spring 自动用这个构造器注入 Bean
}
```

### `@AllArgsConstructor`

为所有字段生成全参构造方法。

```java
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Result<T> {
    private int code;
    private String msg;
    private T data;
    // Lombok 生成: public Result(int code, String msg, T data) { ... }
}
```

## Jakarta Validation（参数校验）

Spring Boot 3.x 使用 Jakarta EE 9+，校验注解的包是 `jakarta.validation.constraints.*`（老版本是 `javax.validation.*`）。

校验加在实体类或 DTO 类的字段上，配合 Controller 方法的 `@Valid` 参数使用。校验失败时抛出 `MethodArgumentNotValidException`。

### `@NotBlank`

字符串不能为 `null` 或全空白字符。

```java
@NotBlank(message = "姓名不能为空")
private String name;
```

### `@Email`

字符串必须是合法的邮箱格式。

```java
@Email(message = "邮箱格式不正确")
private String email;
```

### `@Min` / `@Max`

整数的最小值 / 最大值限制。

```java
@Min(value = 0, message = "年龄不能为负数")
@Max(value = 150, message = "年龄超出合理范围")
private Integer age;
```

### `@NotNull`

字段不能为 `null`。通常用于 `Long`、`BigDecimal` 等包装类型。

```java
@NotNull(message = "金额不能为空")
private BigDecimal amount;
```

### `@DecimalMin`

`BigDecimal` 的最小值校验。

```java
@NotNull(message = "金额不能为空")
@DecimalMin(value = "0.01", message = "金额必须大于0")
private BigDecimal amount;
```

## Spring AOP

### `@Aspect`

声明切面类，和 `@Component` 一起使用交给 Spring 管理。

```java
@Slf4j
@Aspect
@Component
public class ControllerLogAspect {
    // 定义切入点和通知
}
```

### `@Pointcut`

定义切入点表达式，匹配要拦截的方法。

```java
@Pointcut("execution(* com.example.springbootlearning.controller.*.*(..))")
public void controllerMethods() {}
//              ↑ 第一个 * = 任意返回值
//              ↑ controller.* = controller 包下任意类
//              ↑ .*(..) = 任意方法、任意参数
```

### `@Around`

环绕通知，包裹目标方法的执行。必须调用 `joinPoint.proceed()` 并返回其结果，否则原方法不会执行。

```java
@Around("controllerMethods()")
public Object logAround(ProceedingJoinPoint joinPoint) throws Throwable {
    log.info("→ {} | 参数: {}", method, args);   // 方法执行前
    Object result = joinPoint.proceed();          // 执行原始方法
    log.info("← {} | 耗时: {}ms", method, elapsed); // 方法执行后
    return result;
}
```

`@Around` 能拿到参数和返回值，还能控制方法是否执行——其他通知（`@Before`、`@After` 等）都是它的简化版。

### 关键约束

- 切面只对 Spring 管理的 Bean 生效，自己 `new` 的对象不走 AOP
- 同一个类内方法互调不走 AOP（`this` 指向原始对象，绕过了代理）
- `@Around` 不调 `proceed()` 或不返回其结果，功能就断了

## Spring Security

### `@Configuration` + `@EnableWebSecurity`

Spring Security 配置类，两个注解一起用，替代老版本的 `WebSecurityConfigurerAdapter`（已废弃）。

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    // 定义 PasswordEncoder、UserDetailsService、SecurityFilterChain 三个 Bean
}
```

### 核心 Bean 三要素

| Bean | 职责 | 回答的问题 |
|------|------|------------|
| `PasswordEncoder` | 密码加密 | 密码怎么存？ |
| `UserDetailsService` | 加载用户 | 用户从哪来？ |
| `SecurityFilterChain` | 拦截规则 | 哪些路径要保护？ |

```java
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(auth -> auth
            .anyRequest().authenticated()   // 默认全部需要认证
        )
        .httpBasic(withDefaults())
        .csrf(AbstractHttpConfigurer::disable);
    return http.build();
}
```

### `@EnableMethodSecurity`

开启方法级权限控制（默认不开启）。开启后可以在 Controller 方法上直接声明权限：

```java
@PreAuthorize("hasRole('ADMIN')")
@DeleteMapping("/{id}")
public Result<Void> delete(@PathVariable Long id) { ... }
```

### `SessionCreationPolicy.STATELESS`

JWT 模式下必须设为无状态，Spring Security 不会创建 HTTP Session。每个请求独立通过 token 验证。

```java
http.sessionManagement(session ->
    session.sessionCreationPolicy(SessionCreationPolicy.STATELESS)
);
```

### JWT 流程

```
POST /auth/login (公开) → 验证密码 → 返回 JWT
   ↓
后续请求带 Authorization: Bearer <token>
   ↓
JwtAuthFilter (OncePerRequestFilter) → 解析 token → 注入 SecurityContext
```

**关键点**：
- HTTP Basic → 每次带密码；JWT → 只在登录时带密码，后续带 token
- 登录接口 `/auth/login` 必须 `.permitAll()`
- `addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)` 把 JWT 过滤器插到认证链前面
- token 过期或签名不对 → `SecurityContext` 为空 → Spring Security 返回 403
