# 消息队列（RabbitMQ）

## 解决什么问题

转账日志从 `@Async` 线程池改为 RabbitMQ 消息队列。线程池的问题：

- 线程池满了任务直接被丢弃（或抛异常）
- 服务重启，未处理的任务全部丢失
- 生产者和消费者耦合在同一个 JVM 里

消息队列把"发任务"和"做任务"拆成两个独立进程，消息持久化到磁盘，消费者挂了重启后继续处理。

## 核心概念

```
生产者（AccountService）→ Exchange（交换机）→ Queue（队列）→ 消费者（Listener）
```

| 概念 | 说明 |
|------|------|
| **Producer** | 发消息的一方，`RabbitTemplate.convertAndSend()` |
| **Exchange** | 交换机，决定消息投到哪个队列（Direct/Topic/Fanout） |
| **Queue** | 消息排队的地方，持久化到磁盘 |
| **Binding** | Exchange 和 Queue 之间的绑定规则 |
| **Consumer** | 消费消息的一方，`@RabbitListener` 标注 |

本项目用 **DirectExchange**——routing key 完全匹配才投递。

## 配置

### Docker 启动

```bash
docker run -d --name rabbitmq \
  -p 5672:5672 -p 15672:15672 \
  -e RABBITMQ_DEFAULT_USER=admin \
  -e RABBITMQ_DEFAULT_PASS=密码 \
  rabbitmq:management
```

- `5672`：程序连接端口
- `15672`：Web 管理界面

### application.yml

```yaml
spring:
  rabbitmq:
    host: ${RABBITMQ_HOST:localhost}
    port: ${RABBITMQ_PORT:5672}
    username: ${RABBITMQ_USERNAME:admin}
    password: ${RABBITMQ_PASSWORD:}
```

## 声明 Exchange / Queue / Binding

```java
@Configuration
public class RabbitMQConfig {

    public static final String EXCHANGE = "transfer.exchange";
    public static final String QUEUE = "transfer.log.queue";
    public static final String ROUTING_KEY = "transfer.log";

    @Bean
    public DirectExchange transferExchange() {
        return new DirectExchange(EXCHANGE);
    }

    @Bean
    public Queue transferLogQueue() {
        return new Queue(QUEUE, true); // durable=true，重启不丢
    }

    @Bean
    public Binding transferLogBinding() {
        return BindingBuilder.bind(transferLogQueue())
                .to(transferExchange()).with(ROUTING_KEY);
    }
}
```

## 生产者（发消息）

```java
@Transactional
public void transfer(Long fromId, Long toId, BigDecimal amount) {
    accountMapper.deduct(fromId, amount);
    accountMapper.addBalance(toId, amount);
    // 发消息到 RabbitMQ，替代原来的 @Async 方法调用
    TransferLogMessage msg = new TransferLogMessage(fromId, toId, amount);
    rabbitTemplate.convertAndSend(
        RabbitMQConfig.EXCHANGE,
        RabbitMQConfig.ROUTING_KEY,
        msg
    );
}
```

`convertAndSend()` 把 Java 对象自动序列化为 JSON 发送。Spring Boot 默认用 Jackson 序列化。

## 消费者（收消息）

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class TransferLogListener {

    private final TransferLogMapper transferLogMapper;

    @RabbitListener(queues = RabbitMQConfig.QUEUE)
    public void handleTransferLog(TransferLogMessage msg) {
        TransferLog log = new TransferLog();
        log.setFromAccountId(msg.getFromAccountId());
        log.setToAccountId(msg.getToAccountId());
        log.setAmount(msg.getAmount());
        log.setCreateTime(LocalDateTime.now());
        transferLogMapper.insert(log);
    }
}
```

`@RabbitListener` 标注的方法自动监听指定队列，有消息到达时 Spring 自动调用。

## @Async vs RabbitMQ

| | @Async 线程池 | RabbitMQ |
|---|---|---|
| **运行位置** | 同一个 JVM | 独立进程 |
| **任务持久化** | 无（内存中） | 有（磁盘） |
| **服务重启** | 任务丢失 | 消息还在 |
| **消费者扩容** | 改线程池大小 | 加消费者实例 |
| **失败重试** | 无 | 支持（Nack 重回队列） |
| **搭建复杂度** | 一行注解 | 需额外服务 |
| **适用场景** | 轻量、可丢失的任务 | 重要、不可丢的任务 |
