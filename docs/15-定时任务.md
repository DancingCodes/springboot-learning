# 定时任务

## 解决什么问题

有些业务不需要用户触发，而是按时间自动执行——每天凌晨结算利息、每月生成报表、定时清理过期文件。

Spring 的 `@Scheduled` 让你在方法上标注执行时间，框架自动按时调用。

## 核心两个注解

| 注解 | 位置 | 作用 |
|------|------|------|
| `@EnableScheduling` | 启动类 | 告诉 Spring "我要用定时任务"，背后是线程池 + 调度器 |
| `@Scheduled` | 方法上 | 指定这个方法的执行规则（cron、fixedRate 等） |

## Cron 表达式

7 个字段：`秒 分 时 日 月 周`

```
0 0 3 * * ?
│ │ │ │ │ │
│ │ │ │ │ └─ 周（? = 不指定）
│ │ │ │ └─── 月（* = 每月）
│ │ │ └───── 日（* = 每天）
│ │ └─────── 时（3 = 凌晨 3 点）
│ └───────── 分（0）
└─────────── 秒（0）
```

常用示例：

| 表达式 | 含义 |
|--------|------|
| `0 0 3 * * ?` | 每天凌晨 3 点 |
| `0 */5 * * * ?` | 每 5 分钟 |
| `0 0 9 * * MON-FRI` | 工作日 9 点 |
| `0 0 0 1 * ?` | 每月 1 号 0 点 |

## 实战：每日利息结算

每天凌晨 3 点，给所有账户余额结算日息（年化 3% ÷ 365）。

### 利息配置（application.yml）

```yaml
interest:
  annual-rate: 0.03
```

### 启动类加 @EnableScheduling

```java
@SpringBootApplication
@EnableScheduling
public class SpringbootLearningApplication { ... }
```

### Mapper — 批量更新

```java
@Update("UPDATE account SET balance = balance * (1 + #{dailyRate})")
int addInterest(@Param("dailyRate") BigDecimal dailyRate);
```

一条 SQL 更新所有账户，`balance * (1 + 日利率)`，不用逐条查-算-改。

### 定时任务

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class InterestSettlementTask {

    private final AccountMapper accountMapper;

    @Value("${interest.annual-rate}")
    private BigDecimal annualRate;

    @Scheduled(cron = "0 0 3 * * ?")
    public void settleDailyInterest() {
        BigDecimal dailyRate = annualRate.divide(BigDecimal.valueOf(365), 10, RoundingMode.HALF_UP);
        int count = accountMapper.selectCount(null);
        accountMapper.addInterest(dailyRate);
        log.info("利息结算完成 | 年利率: {}% | 日利率: {}% | 影响账户: {}",
                annualRate.multiply(BigDecimal.valueOf(100)),
                dailyRate.multiply(BigDecimal.valueOf(100)).setScale(6, RoundingMode.HALF_UP),
                count);
    }
}
```

### 设计要点

- **新包 `task/`**：定时任务是独立关注点，和 config、service 职责不同
- **利率外置**：`@Value` 注入，改利率不用改代码
- **批量 SQL**：一条 UPDATE 搞定，不走逐条遍历
- **日志在手**：AOP 切面只拦截 `controller.*`，定时任务不会被 AOP 拦截，用 `@Slf4j` 打日志
- **`BigDecimal` 精度**：`divide` 指定 10 位小数 + `RoundingMode.HALF_UP`，避免除不尽抛异常

## @Scheduled 的三种方式

| 方式 | 示例 | 场景 |
|------|------|------|
| `cron` | `@Scheduled(cron = "0 0 3 * * ?")` | 固定时间点（最常用） |
| `fixedRate` | `@Scheduled(fixedRate = 60000)` | 每隔 60s 执行，不管上次是否完成 |
| `fixedDelay` | `@Scheduled(fixedDelay = 60000)` | 上次完成后等 60s 再执行 |

`fixedRate` vs `fixedDelay` 的区别：

```
fixedRate:  开始→(60s)→开始→(60s)→开始
fixedDelay: 开始→结束→(等60s)→开始
```

如果任务是幂等的（执行一次和十次结果一样），用 `fixedRate`。如果必须等上次完成，用 `fixedDelay`。

## 为什么是单线程

Spring 定时任务默认单线程串行执行。多个 `@Scheduled` 方法不会同时跑，一个慢了会堵后面。生产上可以通过配置线程池来并发：

```yaml
spring:
  task:
    scheduling:
      pool:
        size: 4
```

本项目只有一个定时任务，单线程够用。

## 验证定时任务

启动项目后，会看到类似日志（定时任务注册在 INFO 级别可能不打印，DEBUG 才可见）。可以临时把 cron 改成每分钟一次快速验证：

```
@Scheduled(cron = "0 */1 * * * ?")  // 每分钟一次，验证后改回
```

启动后等一分钟，查数据库确认余额增加了。
